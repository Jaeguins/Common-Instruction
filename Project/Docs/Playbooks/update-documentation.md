# Playbook: 문서 갱신

## When To Use
지침/컨텍스트/상태/ADR 등 프로젝트 문서를 갱신할 때 사용한다.
지침·정책 자체의 **변경**은 Governance Change다(`../Governance/README.md`).

## Required Reading
- `../ProjectRules/documentation-policy.md`
- `../Governance/README.md` (지침 변경에 해당하는 경우)

## Procedure
1. 갱신 대상 문서와 변경 성격(사실 갱신 vs 지침 변경)을 구분한다.
2. 사실 갱신이면 해당 문서를 직접 갱신한다.
3. 지침/정책/ADR/절차 변경이면 Governance Change 절차를 따른다.
4. 상위 문서와 중복 설명이 생기지 않게 정리한다.

## Validation
- 문서 내 상대경로가 실제로 존재하는지 확인한다(`../Validation/manual-checklist.md`).
- 갱신 내용이 다른 문서와 모순되지 않는지 확인한다.

## Output
- 갱신된 문서
- 지침 변경이면 `../Governance/`에 변경 제안/기록
