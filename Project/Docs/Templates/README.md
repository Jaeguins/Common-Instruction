# Templates — 디렉토리 로컬 문서 템플릿

이 폴더는 **디렉토리마다 둘 수 있는 로컬 문서**의 범용 템플릿을 모은다.

`Source/`, `Assets/` 같은 디렉토리는 프로젝트의 기술 기반(엔진·언어·구성)에 따라
존재하지 않을 수 있다. 따라서 이런 폴더를 고정으로 만들지 않고, 실제로 디렉토리가 생길 때
여기 템플릿을 **복사해서** 해당 디렉토리 루트에 둔다.

## 포함 템플릿

- `CONTEXT.md` — 디렉토리 탐색용 짧은 지도
- `AGENTS.md` — 디렉토리 로컬 지침
- `ADR-Refs.md` — 디렉토리에 영향을 주는 ADR 참조

## 로컬 문서를 만드는 위치

로컬 문서는 프로젝트의 모든 디렉토리에 만들지 않는다.
반복적으로 에이전트가 혼동하는 디렉토리, 책임이 복잡한 디렉토리, ADR 영향을 받는
디렉토리에만 만든다. (생성 기준은 `/AGENTS.md`의 "로컬 문서 생성 규칙" 참고)

- `CONTEXT.md`: 디렉토리 구조 이해에 반복적으로 도움이 될 때만
- `AGENTS.md`: 상위 규칙과 다른 강한 로컬 규칙이 있을 때만
- `ADR-Refs.md`: 관련 ADR이 실제로 존재할 때만

## 복사 후 수정 체크리스트

```text
- 문서 제목을 실제 경로명으로 변경
- Scope를 실제 경로로 변경
- placeholder 제거
- 관련 ProjectRules 연결
- 관련 Playbooks 연결
- 관련 ADR이 있으면 ADR-Refs.md 생성
- 관련 ADR이 없으면 ADR-Refs.md는 만들지 않음
- 상위 CONTEXT.md와 중복 설명 제거
```

## 주의

- 템플릿 원본은 이 폴더에 유지한다. 복사본만 각 디렉토리에서 수정한다.
- ADR 원문은 항상 `/Docs/Decisions`에만 둔다. 디렉토리에는 `ADR-Refs.md` 참조만 둔다.
- 디렉토리 로컬 지침이 상위 지침과 충돌하면 상위 지침을 우선한다(`/AGENTS.md` 참고).
