# Governance — 지침 변경 절차

이 폴더는 **지침 변경 요청**을 일반 작업 요청과 분리해 처리하는 기준을 담는다.

## 변경 처리 기준

- Governance Change는 지침, 정책, ADR, Playbook, 로컬 문서 구조를 바꾸는 요청이다.
- 일반 기능 구현 요청보다 신중하게 처리한다.
- **대화형 에이전트**는 변경안을 설명하고 사용자 승인 후 반영한다.
- **비대화형 에이전트**는 변경 제안 문서(`CHANGE-TEMPLATE.md` 형식)를 남기고 반영을 보류한다.
- 사용자가 명시적으로 "바로 수정하라"고 한 경우에는 승인된 요청으로 간주한다.
- 명시적 승인 없이 지침/ADR/전역 규칙을 임의로 변경하지 않는다.

## 일반 작업과의 구분

- "이 규칙을 바꿔라", "이제부터 X를 기본으로 해라" → Governance Change
- "이 기능을 구현해라", "이 버그를 고쳐라" → 일반 작업 요청

상세 실행 흐름은 `/AGENTS.md`의 "Governance Change 처리"를 따른다.
