# ProjectRules — 안정 전역 규칙

이 폴더는 프로젝트 전역에서 안정적으로 유지되는 규칙을 담는다.
자주 바뀌는 상태는 여기 두지 않는다(→ `../ProjectState/`).

## 규칙 작성 기준

- 각 문서는 아래 공통 형식을 따른다.
- 현재 프로젝트 고유 내용을 확정해 쓰지 않는다. "해당 프로젝트에서 사용하는 방식을 여기에 작성한다"로 둔다.
- 예시는 placeholder로만 남긴다.
- "반드시 X를 사용한다" 같은 확정 표현 대신, 규칙을 적을 **위치와 형식**을 제공한다.

## 공통 형식

```md
# 문서 제목

## Purpose
이 문서가 담당하는 범위를 설명한다.

## Stable Rules
프로젝트 전역에서 안정적으로 유지되어야 하는 규칙을 작성한다.

## Allowed Exceptions
예외가 가능하다면 어떤 기준으로 허용하는지 작성한다.

## Validation
이 규칙이 지켜졌는지 확인하는 방법을 작성한다.

## Related Documents
관련 ProjectRules, Playbooks, ADR 경로를 작성한다.
```

## 포함 문서 (Core)

기본 템플릿에는 모든 프로젝트에서 거의 공통으로 쓰는 최소 규칙만 둔다.

- `coding-convention.md` — 코드 스타일 원칙
- `architecture-policy.md` — 구조 설계 원칙
- `testing-policy.md` — 테스트 기준
- `performance-policy.md` — 성능 판단 기준
- `documentation-policy.md` — 문서 갱신 기준

## 프로젝트별 규칙 추가 기준

프로젝트에 UI, 로컬라이징, 네트워크, 저장, 데이터테이블, 플랫폼별 빌드, 에셋 번들 등
특수 영역이 생기면 해당 규칙 문서를 **그때 추가**한다.
기본 템플릿은 이런 영역을 미리 생성하지 않는다(엔진/플랫폼 전제를 줄이기 위함).

새 규칙 문서도 위 "공통 형식"을 따른다.
